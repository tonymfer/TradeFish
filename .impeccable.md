# TradeFish — Impeccable Design Context

This file captures the design context for the TradeFish project, synthesized from the
shipped design system at `/Users/tomo/Downloads/TradeFish` (which includes a full
SKILL.md and README.md). It overrides default impeccable heuristics where the brand has
intentionally chosen otherwise (see the "Brand-overrides-defaults" section).

## Design Context

### Users

- **Primary**: Crypto-native traders and trading-agent builders attending an on-chain
  hackathon and exploring "TradeFish" as a live demo. They evaluate fast, scan numbers
  first, and respect terminals more than dashboards.
- **Secondary**: Hackathon judges and sponsor reps (Flock, Nansen, Virtuals,
  PancakeSwap, BananaGun, Base) — they need to grok the value prop in 10 seconds, see
  named integrations, and feel the surface is real, not a slideshow.
- **Context of use**: Desktop-first viewing of the demo URL during a 2-minute pitch.
  Mobile is supported but not optimized — the canonical surface is the 4-panel quadwork
  arena terminal on a wide screen.

### Brand Personality

- Three words: **operator-terse, phosphor-warm, accountable.**
- Voice: Bloomberg ticker scripted by a sysadmin. Direct, declarative, no marketing
  softness. System-voice over second-person ("ROUND #1,247 · ELAPSED 00:42", not "your
  round is starting").
- Numbers are first-class — always tabular-nums, explicit signs and units (`+$184.20`,
  `-$21.80`, `64,212`, `12 GWEI`).
- Casing: ALL CAPS for labels/status/nav, Title Case for proper nouns, lower-case for
  agent chat dialogue, snake_case / DOT.NOTATION for system identifiers.
- Punctuation language: `▸` to lead, `·` between fields, `┌─` and `└─` in panel
  headers, `●` for live, `◆◇` for tier glyphs.

### Aesthetic Direction

- **Visual world**: cool-black ocean — `#050a14` body, deep navy panel fills, cyan
  `#a8d8e8` as the scarce hero accent reserved for live/active/consensus/primary CTA.
  No light themes. Ever.
- **Type**: Departure Mono only. One typeface. Pixel/display/sans CSS aliases all
  point to it; aliases exist so future role swaps don't ripple.
- **Corners**: Sharp, `--r-0: 0`. Single exception is `--r-pill` for live-indicator
  dots and bubble-map nodes.
- **Texture stack** (back to front): solid `--bg-0` → 24×24 dot lattice masked to
  center ellipse → vignette to `rgba(0,0,0,0.65)` at edges → CRT scanlines (4px
  repeating, `mix-blend-mode: overlay`, ~2.5% alpha).
- **Iconography**: NO icon font, NO emoji. Unicode glyphs and box-drawing chars
  (`◆ ◇ ▸ · ● ┌─ └─ → ↵`). Sponsor marks are TEXT in caps (NANSEN, BNGUN, PCS, FLOCK).
- **Imagery**: zero photography, zero gradients except the sub-5% panel header
  (`linear-gradient(180deg, rgba(168,216,232,0.04), transparent)`). The brand's
  imagery is data viz: bubbles, sparklines, orderbooks, ASCII tables.
- **Motion**: linear or `steps(1)` only. Two speeds — `--t-fast 150ms` for hover,
  `--t-med 280ms` for state changes. No bounces, no spring physics. Things tick,
  blink, snap. Caret blink, phosphor flicker, live-dot pulse.
- **Hover/press**: border `--line` → `--line-strong`, text `--fg-dim` → `--cream`,
  no scale. Active buttons invert (`background: cream; color: bg-0`).
- **Glow** instead of drop shadows: `box-shadow: 0 0 0 1px <color, 0.45>, 0 0 20px
<color, 0.25>`. Used on active CTA, consensus core, tick-up flashes only.
- **Layout DNA**: 8px spacing grid; quadwork-style 4-panel grids with 1px gutters
  via `gap: 1px` over a `--line` background (the canonical app layout for the arena);
  fixed 44px top bar + 36px status bar with `backdrop-filter: blur(8px)`.
- **Trading semantics**: `--long #7fe0a8` (phosphor green up), `--short #e07560`
  (coral red down), `--hold #ffb84a` (amber neutral). Use the `-bg` 8–10% alpha tints
  for cell backgrounds. Reserved — never use these hues decoratively.

### Anti-references

- No emoji. No Lucide / Heroicons / icon fonts. No rounded corners (except dots).
- No glassmorphism for its own sake. Background blur is allowed only for the fixed
  top/status bars, which already follow the brand pattern.
- No proportional fonts. Inter, Roboto, DM Sans, Plex etc. are all banned.
- No drop shadows. Phosphor glow only.
- No gradients except the sub-5% panel header tint.
- No "AI dashboard" tropes: side-stripe accent borders, gradient text, hero metric
  templates with big purple-cyan numbers. The terminal aesthetic intentionally
  avoids the AI-slop palette.

### Design Principles

1. **Numbers earn their pixel.** Every figure is tabular-nums, signed, unit-bearing,
   and visible. The interface should read like an L1 block explorer crossed with a
   1990s Bloomberg terminal.
2. **Constrain to amplify.** One typeface, one palette, sharp corners, unicode
   icons. The surface looks distinctive because it refuses to soften itself.
3. **Cyan is rare.** Most pixels are `--fg-dim` over deep navy. Cyan means _live,
   active, consensus, primary CTA_ — and nothing else. Dilute it and the brand dies.
4. **Motion ticks, doesn't ease.** `linear` and `steps(1)`. Soft easings break the
   CRT/terminal illusion.
5. **Quadwork is the canonical layout.** The arena is a 4-panel grid with 1px gutters.
   Every screen should compose from terminal-chrome panels (`tf-term` / `tf-card`),
   never from soft cards with shadows.

## Brand-overrides-defaults

The following impeccable defaults are intentionally overridden by the TradeFish brand —
do not "correct" toward generic taste:

| impeccable default                                                     | TradeFish override                              | Why                                                                                     |
| ---------------------------------------------------------------------- | ----------------------------------------------- | --------------------------------------------------------------------------------------- |
| "DO NOT use monospace as lazy shorthand for technical/developer vibes" | Departure Mono mono everywhere, single typeface | Brand identity is a 1990s pixel-mono terminal. The font IS the brand.                   |
| "Pair a distinctive display font with a refined body font"             | One typeface only                               | "TradeFish has no proportional fonts." Pair would dilute the terminal illusion.         |
| "Theme should be derived from audience and viewing context"            | Always dark `--bg-0 #050a14`                    | Brand mandate: cool-black ocean only, no light themes.                                  |
| "DO NOT default to dark mode with glowing accents"                     | Phosphor glow on active CTA + consensus core    | Glow is a deliberate CRT-phosphor signal, used scarcely on `--cyan`/`--long`/`--short`. |
| "Avoid neon accents on dark backgrounds"                               | Cyan `#a8d8e8` as scarce live/active accent     | Cyan is the single brand accent and is reserved (≤10% of pixels).                       |
| "Use rounded radii for warmth"                                         | Sharp pixel corners (`--r-0: 0`) everywhere     | Sharp corners are non-negotiable per SKILL.md hard rule #3.                             |
